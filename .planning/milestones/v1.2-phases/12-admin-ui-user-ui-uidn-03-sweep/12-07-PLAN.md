From src/components/ui/button.tsx — shadcn Button (new-york style) supports asChild via Radix Slot:
```tsx
function Button({
  className,
  variant,
  size,
  asChild = false,
  ...props
}: React.ComponentProps<"button"> &
  VariantProps<typeof buttonVariants> & {
    asChild?: boolean
  }) {
  const Comp = asChild ? Slot : "button"
  return (
    <Comp
      data-slot="button"
      className={cn(buttonVariants({ variant, size, className }))}
      {...props}
    />
  )
}
```

When `asChild` is set, the Button forwards its className/data-slot/styling onto its single child element. Wrapping a TanStack `<Link to="/admin">` produces `<a href="/admin" class="...ghost variant classes...">Cancel</a>`.

From src/components/suggestions/form/SuggestionForm.tsx — Link is already imported on line 2:
```tsx

```

The header back-link in the same file (lines 144–149 in the loadError branch, lines 166–171 in the main return) already uses `<Link to="/admin">…</Link>`. The Cancel button (line 277) is the only remaining native-button back-navigator.

Current Cancel button (the regression — to be replaced):

```

```

Note: the actual existing button uses `variant="ghost"`, not `variant="outline"` as the gap brief example showed. Preserve `ghost` so the visual contract matches pre-regression behavior.
