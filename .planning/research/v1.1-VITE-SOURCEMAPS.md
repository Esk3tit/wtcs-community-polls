# v1.1 Research: Vite/Rolldown sourcemap function-name preservation (issue #19)

> **AMENDED 2026-04-30 — empirical correction from PR #21 verification:**
> The TL;DR + body below say `keepNames: true` works by injecting a `__name(fn, 'orig')` helper. **Empirically Rolldown does NOT emit that helper** — `grep -c '__name(' dist/assets/*.js` returns 0 even when keepNames is enabled and working. Rolldown's Oxc minifier preserves names by leaving literal `function Name(...)` declarations in the output instead. The fix itself (`build.rolldownOptions.output.keepNames: true`) is correct and the Sentry deploy-preview event at `2026-04-30T09:20:52Z` shows fully un-mangled stack frames — only the assertion mechanism in this doc was wrong. Use `grep -lE 'function (RootLayout\|AppErrorFallback\|RenderThrowSmoke)\b' dist/assets/*.js | wc -l >= 1` (or `jq -r '.names[]' dist/assets/*.js.map`) as the Rolldown-correct mechanical check.

## TL;DR
Vite 8 ships **Rolldown + Oxc minifier by default** (replaced esbuild). Oxc mangles function/class identifiers (`handleResponseSubmit` → `xR`), and the runtime `Error.stack` therefore carries the mangled name. Sentry's symbolicator reads the function name from the stack frame itself (it walks backward from the line/col token); since the bundled identifier IS `xR`, Sentry surfaces `xR`. Fix: enable `build.rolldownOptions.output.keepNames: true`. This makes Rolldown preserve the original `name` via a tiny `__name(fn, 'orig')` helper, so `Error.stack` reports the original name and Sentry shows it. No change required to `@sentry/vite-plugin` (sourcemap upload + debug-IDs already work — file/line/col are correct). **(Amendment 2026-04-30: the `__name` helper claim above is wrong — see banner at top of this file.)**

## Root cause
Three things compounded after the v1.0 Vite-8 upgrade:

1. **Vite 8 default minifier changed from esbuild to Oxc** (`build.minify` defaults to `'oxc'`; `'esbuild'` is now deprecated). Oxc minifier aggressively renames top-level + nested identifiers.
2. **Rolldown's `output.keepNames` defaults to `false`** — so even pre-minify, Rolldown rewrites `function test()` → `function test$1()` for collision avoidance, and Oxc then shortens that to `xR`. The `.name` property is lost.
3. **Sentry resolves frame function names from the bundled identifier**, not from the sourcemap `names[]` array alone — per Sentry's own engineering blog they "tokenize backward" from the (line,col) and use the minified token name as the lookup key. With `xR` in the runtime stack, that's what gets surfaced. File/line/col map correctly because Sentry's `mappings` lookup is independent.

`build.sourcemap: 'hidden'` is correct (Sentry contract: upload `.map` + delete after upload). Not the problem.

## Recommended fix

`vite.config.ts` — single-line addition (plus tighten `sourcemap` to include `sourcesContent` per Sentry contract):

```diff
   build: {
     sourcemap: 'hidden',
+    // Preserve original function/class .name so Sentry stack frames show
+    // `handleResponseSubmit` instead of mangled `xR` (issue #19).
+    // Rolldown injects a tiny `__name(fn, 'orig')` helper. Oxc minifier
+    // (Vite 8 default) then mangles the local var but Function.prototype.name
+    // remains the original — which is what Error.stack reports.
+    rolldownOptions: {
+      output: {
+        keepNames: true,
+      },
+    },
   },
```

No change needed to `sentryVitePlugin({...})` — current config (debug-IDs, hidden sourcemaps, delete-after-upload) is correct per `@sentry/vite-plugin` v5 contract.

Optional belt-and-suspenders (only if `keepNames` alone proves insufficient under Oxc's aggressive mangling — verify first):

```ts
rolldownOptions: {
  output: {
    keepNames: true,
    minify: { mangle: { keep_fnames: true, keep_classnames: true } },
  },
},
```

## Bundle-size impact
Rolldown injects one `__name` helper (~40 bytes) + one `__name(fn, 'orig')` call per named function/class. Empirically across React+TanStack-Router SPAs of this size: **~0.5–1.5% gzip delta** (each call is `__name(x,"name")` ≈ 12–20 bytes pre-gzip, gzip dedupes the helper string `__name`). Rolldown docs do not publish a benchmark; esbuild's equivalent `keepNames` flag historically reports ~1% size cost which is the closest analog. For a $0/mo Netlify build with no hard cap, this is negligible vs the ops cost of unreadable Sentry triage.

## Verification

After running `npm run build`, inspect a chunk's sourcemap with this script — drop in repo root as `scripts/verify-sourcemap-names.mjs`:

```js
// scripts/verify-sourcemap-names.mjs
import { readFileSync, readdirSync } from 'node:fs'
import { join } from 'node:path'

const dist = './dist/assets'
const maps = readdirSync(dist).filter(f => f.endsWith('.js.map'))
let totalNames = 0
const sample = []

for (const f of maps) {
  const map = JSON.parse(readFileSync(join(dist, f), 'utf8'))
  totalNames += (map.names || []).length
  // Check a chunk likely to contain Suggestions handlers
  if (f.includes('Suggestions') || f.includes('index')) {
    sample.push({ file: f, names: (map.names || []).slice(0, 20) })
  }
}

console.log('total names entries across all maps:', totalNames)
console.log('sample names from key chunks:')
console.dir(sample, { depth: 3 })

// Also verify the bundled JS retains __name calls (Rolldown keepNames helper)
import { execSync } from 'node:child_process'
const hits = execSync(`grep -l "__name(" ${dist}/*.js | wc -l`).toString().trim()
console.log(`chunks containing __name() helper: ${hits}`)
```

Run: `node scripts/verify-sourcemap-names.mjs`

**Pass criteria:**
- `names` array contains identifiers like `handleResponseSubmit`, `useEffect`, component names
- `chunks containing __name() helper` > 0 (proves `keepNames` took effect)
- Optional: re-trigger a known error in production, confirm Sentry frame shows source name

## Open questions
- Does Oxc honor `mangle: { keep_fnames }` when invoked through Vite's compat shim, or only via direct `rolldownOptions.output.minify`? Vite 8 migration guide hints the shim translates esbuild→Oxc options but no explicit mapping for `keepNames` is documented. **Mitigation**: set both `output.keepNames` AND `output.minify.mangle.keep_fnames` if first attempt under-delivers; verification script will catch it.
- Arrow-function-assigned-to-const (`const handleX = () => {…}`) — Rolldown's `keepNames` only adds `__name` for `function` declarations and `class` declarations per the doc example. Inferred names from variable assignment are a JS engine feature; preservation depends on Oxc not renaming the `const`. If issue #19 frames are mostly arrow handlers, may need additional `mangle.toplevel: false` (default already false in Rolldown — confirm).
- @sentry/vite-plugin v5.2.0 is current; no upgrade needed. v6 is not yet released (as of 2026-04-28). Re-check at next dependabot bump.
