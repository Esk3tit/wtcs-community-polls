# v1.1 Research Summary — Hygiene & Polish

## Milestone Context

v1.1 is a hygiene/polish milestone on top of the shipped v1.0 (live at polls.wtcsmapban.com since 2026-04-28). It closes carry-forward debt — UI design closure evidence, observability robustness, E2E test fixture/seed isolation, and planning artifact backfill — so the project is audit-clean before v1.2 (SEED-002 admin-controlled per-suggestion results visibility) opens. No new product features in scope; phase numbering continues from v1.0 (last phase 6) → starts at Phase 7.

## TL;DR — One Sentence Per Issue

- **#17 Sentry ErrorBoundary:** Wire `Sentry.reactErrorHandler()` into `createRoot`'s `onUncaughtError` / `onCaughtError` / `onRecoverableError` (React 19 routes caught render errors here, not through `ErrorBoundary`); keep `ErrorBoundary` for fallback UI + add a manual `captureException` belt in `onError`.
- **#19 Vite sourcemaps:** Add `build.rolldownOptions.output.keepNames: true` in `vite.config.ts` so Rolldown injects `__name(fn,'orig')` and `Function.prototype.name` survives Vite 8's new Oxc minifier — Sentry frames recover original names with no `@sentry/vite-plugin` change.
- **#11/#12/#13 Playwright:** Adopt rule **E2E-SCOPE-1** — every list locator filters to `[E2E]`-prefixed cards via `Locator.filter({ hasText: /^\[E2E/ })` before `count`/`first`/`nth`, enforced by a one-line ESLint `no-restricted-syntax` rule. #11: comment-only (already fixed in 7ef6c82); #12: scope locator + cast a vote in-test, tighten regex to `[1-9]\d*`; #13: scope `cards` to `[E2E]` filter so `toHaveCount(1)` is honest.
- **#18 Mobile/shadcn audit:** One-off Lighthouse CLI run (`--form-factor=mobile`) per route + 6-width (320/375/414/768/1024/1440) Playwright screenshot matrix for UIDN-02; per-route 12-item shadcn/Maia/Neutral compliance checklist for UIDN-03 — both archived under `.planning/closure/artifacts/`.

## Phase Boundaries (suggested for roadmapper)

- **Phase 7 — Observability hardening (#17 + #19):** Both are config/wiring changes localized to `src/main.tsx` + `vite.config.ts`. They share one Netlify deploy-preview symbolication smoke pass (throw a render-phase error via `?sentry-render-smoke=1`, confirm Sentry receives event with un-mangled frame names + populated `componentStack`). Tight bundle, fast verify.
- **Phase 8 — E2E test hygiene (#11 + #12 + #13):** Land convention E2E-SCOPE-1 + ESLint rule first (sub-phase A), then per-spec edits (sub-phase B). Optional sub-phase C: extract `freshPoll` Playwright test-scoped fixture (~1 day) to kill the bug class for future specs. Hybrid SQL-seed + Playwright-fixture model retained.
- **Phase 9 — UI closure evidence (#18):** Lighthouse + breakpoint matrix → UIDN-02; per-route shadcn checklist + drift fixes → UIDN-03. Both produce `.planning/closure/UIDN-0{2,3}-*.md` evidence files that flip PROJECT.md Key Decisions from "Revisit" to "Good". **Blocked on user decision** for the shadcn style discrepancy below.
- **Phase 10 — Planning hygiene backfill (no research dep):** VALIDATION.md frontmatter on phases 01–04, Phase 03 VERIFICATION.md retro, 17 SUMMARY frontmatter `requirements-completed` declarations, Phase 04 UAT 6a evidence backfill, Phase 03 UAT 2+3 with second human (no 2FA). Pure docs/metadata — can run in parallel with Phase 7–9 if desired.

## Cross-Cutting Decisions

- **One Netlify deploy-preview smoke pass covers Phase 7.** The `?sentry-render-smoke=1` Playwright check should assert un-mangled function names in the captured frame — verifies #17 (capture path) and #19 (symbolication) in a single round-trip.
- **E2E-SCOPE-1 is a permanent project rule, not a one-time fix.** Add to repo conventions doc + enforce via ESLint so future specs cannot regress.
- **Closure evidence files live under `.planning/closure/`** with PNG/HTML/JSON artifacts in `.planning/closure/artifacts/` — not committed to `dist/` or `public/`.

## Notable Risks & Open Questions

- **shadcn style discrepancy (USER DECISION REQUIRED before Phase 9):** `components.json` declares `style: "new-york"` but `.planning/DESIGN-SYSTEM.md` claims `Maia`. Components are already shipped — most likely fix is to update DESIGN-SYSTEM.md to reflect new-york, but the user must confirm canonical style before UIDN-03 can sign off. Do NOT restyle components in v1.1 (scope creep).
- **#17 mechanism tag uncertainty:** `reactErrorHandler` may surface as `mechanism.type === 'react.errorboundary'` OR `'generic'` — confirm on a deploy preview before locking the Playwright assertion.
- **#19 arrow-function handlers:** Rolldown's `keepNames` documents `function`/`class` declaration support but not inferred names from `const handler = () => {…}`. Verification script must spot-check arrow handlers; fallback is `output.minify.mangle.keep_fnames: true`.
- **#18 auth-gated routes for Lighthouse:** `/admin/*` requires Discord login. Decide: run as guest (capture redirect) or supply session via `--extra-headers`.
- **Lighthouse CI deferred to v1.2:** Free-tier GH Actions minutes are scarce; one-off CLI committed under `.planning/closure/` is the v1.1 answer.

## Out of Scope (explicit non-goals for v1.1)

- New product features (deferred to v1.2 — SEED-002 admin-controlled visibility)
- Lighthouse CI in GitHub Actions (defer to v1.2; honor $0 budget)
- Full migration to Playwright fixtures (hybrid SQL + test-scoped fixture kept; full migration is opportunistic in Phase 8 sub-phase C)
- shadcn style migration (only reconcile DESIGN-SYSTEM.md vs components.json — do not restyle components)
- `polls.is_test` DB CHECK constraint (would make E2E-SCOPE-1 machine-enforceable on the seed side; defer — adds schema churn)
