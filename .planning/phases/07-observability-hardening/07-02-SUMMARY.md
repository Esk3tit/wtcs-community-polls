## Decisions Made

- **Round-2 review HIGH-2 (deterministic inspection build):** Task 3 uses `npx vite build --mode development` so the Sentry plugin's `disable: mode !== 'production'` guard short-circuits and we don't trigger upload-and-delete for a local-only check.
- **Round-3 consensus HIGH (curl assertion vacuous):** Replaced Round-2's curl-based gate verification with a real-browser DevTools checkpoint. All three Round-3 reviewers (Gemini, Codex, Cursor) flagged the curl approach as a false-positive generator for SPA route gates because curl never executes JS and TanStack `beforeLoad` runs only after JS executes.
- **Browser checkpoint executed via Playwright MCP** (user-requested) rather than manual human verification. The Playwright MCP server provided real-browser DOM + network introspection without adding `@playwright/test` to the codebase (Phase 8 = E2E test hygiene per ROADMAP — `@playwright/test` already exists for that scope and Phase 7 deliberately stays out of scope).
