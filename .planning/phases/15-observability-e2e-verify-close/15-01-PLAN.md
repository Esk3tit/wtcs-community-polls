Extend the existing prod-gated `/__smoke` route with two distinct render-phase throw triggers (`?fire=render` and `?fire=dedupe`) so OBSV-05's Dedupe smoke can fire DISTINCT error messages per scenario, and surface `Sentry.lastEventId()` via both `console.log` (human verifier) and `document.body.dataset.sentryEventId` (future automation).

Architecture invariant (from cross-AI plan review HIGH-2): a `useEffect` inside `SmokePage` cannot reliably read `Sentry.lastEventId()` because the app-level `Sentry.ErrorBoundary` in `src/main.tsx` replaces the entire router subtree with `<AppErrorFallback />` when the render throw fires — `SmokePage` is unmounted, so its `useEffect` may never run on a direct visit. The reliable surface is a LOCAL `Sentry.ErrorBoundary` INSIDE `SmokePage` that wraps the throw component. Its `onError(error, componentStack, eventId)` callback receives the event ID directly as the third argument and runs synchronously before the outer boundary can re-render the page, so the dataset write is deterministic.

    **Architecture switch (cross-AI replan HIGH-2 fix):** Do NOT use a `useEffect` inside `SmokePage`. The app-level `Sentry.ErrorBoundary` in `src/main.tsx` catches the render throw and replaces the entire router subtree with `<AppErrorFallback />`, unmounting `SmokePage` before its effect can run. Instead, wrap the throw component in a LOCAL `Sentry.ErrorBoundary` defined inline inside `SmokePage`, and surface the event ID from its `onError` callback (which receives `eventId` as its third argument synchronously when the boundary catches).

    Concrete shape:

    ```tsx
    import { lazy, Suspense, useCallback } from 'react'
    import { createFileRoute, notFound } from '@tanstack/react-router'
    import * as Sentry from '@sentry/react'
    // ...existing lazy import(s) + extended ones from Task 1...

    function SmokePage() {
      const { render, fire } = Route.useSearch()

      // Surface the captured event ID for human verifiers (DevTools console)
      // and future automation (DOM dataset). Lives on the LOCAL boundary's
      // onError so SmokePage is not unmounted before the write — the
      // app-level boundary in src/main.tsx would replace this whole subtree.
      const surfaceEventId = useCallback(
        (_error: unknown, _componentStack: string, eventId: string) => {
          // eslint-disable-next-line no-console -- intentional eventId surface for smoke verifier
          console.log('[smoke] sentry.eventId', eventId)
          document.body.dataset.sentryEventId = eventId ?? ''
        },
        []
      )

      // Inline fallback: keep SmokePage mounted (so the local boundary's
      // onError runs to completion) and visible (so the verifier knows the
      // throw fired without reloading the whole router subtree).
      const SmokeFallback = () => (
        <p className="text-sm text-muted-foreground">Smoke throw captured.</p>
      )

      if (fire === 'render' || fire === 'dedupe') {
        const message = fire === 'render' ? 'OBSV-03 render' : 'OBSV-05 dedupe'
        return (
          <Sentry.ErrorBoundary fallback={<SmokeFallback />} onError={surfaceEventId}>
            <Suspense fallback={<p className="text-sm text-muted-foreground">Loading smoke component…</p>}>
              <RenderThrowSmoke message={message} />
            </Suspense>
          </Sentry.ErrorBoundary>
        )
      }

      // ...existing `?render=1` canary branch + default instructional branch from Task 1...
    }
    ```
