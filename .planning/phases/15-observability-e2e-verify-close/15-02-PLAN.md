<objective>
Create `scripts/verify-sourcemap-names.mjs` — a zero-dependency Node ESM script that reads `dist/assets/*.js` after a production build and asserts every allowlisted component name appears as a literal `function Name(` declaration. Exits non-zero on misses with a structured failure message. Phase 15 wires this script into CI in Plan 03; this plan creates only the script.

Purpose: Permanent regression guard for `vite.config.ts` `rolldownOptions.output.keepNames: true`. If the flag is dropped or Rolldown changes its minification behavior, Sentry stack frames silently degrade to mangled identifiers (`$M`) — without this gate, the degradation is invisible until someone tries to debug a production error.

Output: `scripts/verify-sourcemap-names.mjs` exists at repo root under a new `scripts/` directory; runs against `dist/assets/*.js`; exits 0 on success, non-zero with diagnostic output on failure.
</objective>

    Imports: pure Node — `import { readdir, readFile, access } from 'node:fs/promises'` and `import { join } from 'node:path'`. NO npm dependencies, NO Playwright, NO Supabase.

    Define a top-level `const ALLOWLIST = [...]` containing the locked list from Task 1. Define `const DIST_ASSETS = 'dist/assets'` as the search root (relative to repo root — script is expected to be invoked from repo root by CI and by the developer).

    Body logic:
    1. Verify `dist/assets/` is reachable via `access(DIST_ASSETS)`. If it throws ENOENT, emit a structured error (`console.error`) telling the user to run `npm run build` first, then `process.exit(1)`. The catch block MUST log and exit non-zero (no empty catch).
    2. `readdir(DIST_ASSETS)`, filter to entries ending in `.js`.
    3. Read all `.js` files (via `Promise.all` over `readFile(..., 'utf8')`), join their contents into one string. Acceptable to read each separately and concatenate — memory cost is bounded by the production bundle size (under 5 MB).
    4. For each name in `ALLOWLIST`, build a `new RegExp(\`\\\\bfunction \${name}\\\\b\`)` (literal `function `, the name, a word boundary) and test against the combined content. The `\\b` boundary prevents `functionNameInferred` false positives.
    5. Collect names where the regex did NOT match into `const missing = [...]`.
    6. On `missing.length > 0`: emit a structured failure block via `console.error`:
       - heading line: `[verify-sourcemap-names] FAIL: <N> name(s) missing from dist/assets/*.js`
       - bulleted lines: `  - <name>`
       - actionable resolution sentence: `This means vite.config.ts rolldownOptions.output.keepNames may have regressed. Restore the flag or update the allowlist if a name was renamed in source.`
       - `process.exit(1)`
    7. On `missing.length === 0`: emit a single `console.log` success summary line including: scanned file count, allowlist size, and a one-line statement that the `keepNames` contract holds. Process exits 0 by falling off the end.

    Constraints:
    - Use 2-space indentation.
    - Use `const` / `let` (not `var`); avoid mutation where possible.
    - No `try/catch` wrappers around the main flow except for the `access(DIST_ASSETS)` probe (which IS expected to fail when `dist/` is absent). Any other catch block must log and exit/rethrow.
    - The script file MUST be marked executable in spirit (no shebang needed — invoked via `node scripts/verify-sourcemap-names.mjs`). Do NOT add a shebang line (the analog `.planning/closure/audit-screenshots.mjs` has none).
    - No `eslint-disable` directives needed — `eslint.config.js` already excludes `scripts/` via its default project layout, but confirm by running `npm run lint` after writing.

    The file MUST exit 0 in the green path with no extra noise on stderr.
  </action>
