E2E-SCOPE-1 rule (from Plan 02 — eslint.config.js):
- Fires on: `.toHaveCount/.first/.nth/.last/.all` calls without a preceding `.filter()` anywhere in the SAME CALL EXPRESSION.
- Compliant pattern: `.filter({ hasText: /\[E2E\]/ }).first()` OR `.filter({ hasText: someValue }).first()` (any `.filter()` in the same chain satisfies the AST walk per D-07).
- AST limitation (REVIEWS R-1): the rule cannot trace variable assignments. `const cards = page.locator('x').filter(...); cards.first()` WILL fire because `cards.first()` has no `.filter()` descendant in its own AST. Two mitigation options per call site:
  - (a) inline the filter into every chain (verbose but rule-clean) — chosen for `filter-search.spec.ts`
  - (b) `// eslint-disable-next-line no-restricted-syntax -- WHY` (D-06; use only when locator is already DOM-scoped) — chosen for `browse-respond.spec.ts` L32/L40
