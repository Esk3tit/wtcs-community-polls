    ```yaml
    name: CI

    on:
      pull_request:
        branches: [main]
      push:
        branches: [main]

    concurrency:
      group: ci-${{ github.ref }}
      cancel-in-progress: true

    jobs:
      lint-and-unit:
        runs-on: ubuntu-latest
        timeout-minutes: 10
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with:
              node-version: '22'
              cache: 'npm'
          # D-16 §2 — npm ci, never npm install. Fails if lockfile and package.json disagree.
          - run: npm ci
          - run: npm run lint
          - run: npm test -- --run
          # D-16 §4 — non-blocking supply-chain check.
          - name: npm audit (non-blocking)
            run: npm audit --audit-level=high || true

      e2e:
        runs-on: ubuntu-latest
        timeout-minutes: 15
        needs: lint-and-unit
        env:
          # Local-only fixed-default keys from `supabase start`. Safe to store as repo secrets.
          VITE_SUPABASE_URL: http://localhost:54321
          VITE_SUPABASE_ANON_KEY: ${{ secrets.LOCAL_ANON_KEY }}
          SUPABASE_SERVICE_ROLE_KEY: ${{ secrets.LOCAL_SERVICE_ROLE_KEY }}
        steps:
          - uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with:
              node-version: '22'
              cache: 'npm'
          - run: npm ci
          - uses: supabase/setup-cli@v1
            with:
              version: 2.92.1
          - name: Start Supabase local stack
            run: supabase start
          - name: Wait for stack ready
            run: |
              for i in {1..30}; do
                if supabase status 2>/dev/null | grep -q 'API URL'; then
                  echo "Supabase ready after ${i}s"
                  exit 0
                fi
                sleep 2
              done
              echo "Supabase did not become ready in 60s"
              supabase status
              exit 1
          - name: Apply fixture seed
            run: |
              DB_URL=$(supabase status | grep 'DB URL' | awk '{print $3}')
              psql "$DB_URL" -f e2e/fixtures/seed.sql
          - name: Install Playwright browsers
            run: npx playwright install --with-deps chromium
          - name: Build app
            run: npm run build
          - name: Start preview server
            run: npx vite preview --host 0.0.0.0 --port 4173 &
            shell: bash
          - name: Wait for preview
            run: |
              for i in {1..30}; do
                if curl -sf http://localhost:4173 >/dev/null; then
                  echo "Preview ready"; exit 0
                fi
                sleep 1
              done
              exit 1
          - name: Run Playwright smoke suite
            run: npx playwright test --grep @smoke
            env:
              PLAYWRIGHT_BASE_URL: http://localhost:4173
          - name: Upload Playwright report on failure
            if: failure()
            uses: actions/upload-artifact@v4
            with:
              name: playwright-report
              path: playwright-report/
              retention-days: 7
    ```

    Split into two jobs — `lint-and-unit` (fast) and `e2e` (slower, Supabase-
    dependent). The E2E job gates on lint-and-unit success via `needs:`.

    Do NOT pin third-party GH Actions beyond major (`@v4`) — Dependabot (Plan
    05-07) will promote them on a schedule.

<security>
## Threats Mitigated

| Threat ID | Description | Mitigation |
|-----------|-------------|------------|
| T-05-11-cron-secret-in-logs | Secrets echoed to workflow logs | Uses `${{ secrets.* }}` references (auto-masked); no `echo "$SECRET"`; `|| true` on `npm audit` avoids noisy log capture that could incentivize grep-to-diagnose workflows |

## Residual Risks
- `LOCAL_ANON_KEY` / `LOCAL_SERVICE_ROLE_KEY` are fixed-default `supabase start` keys — not sensitive, but stored as "secrets" for consistency. Acceptable.
- `npm audit` non-blocking means a HIGH advisory shows up as a green run. Deliberate per D-16 §4 ("initial threshold is non-blocking warning to avoid breaking launch"). Promotable later.
</security>
