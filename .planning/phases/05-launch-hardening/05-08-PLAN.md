<rollback>
## M6 Rollback Runbook

Two contingencies are explicitly handled — if either fires, execute the steps
below instead of proceeding. Both rely on the D-11 dual-registration in Task
4 (both Netlify default URL and `polls.wtcsmapvote.com` allowlisted in
Discord), which means we can serve under either host without code changes.

### Contingency A — Discord OAuth callback fails post-CNAME flip

Symptoms: user clicks "Sign in with Discord" on `polls.wtcsmapvote.com`,
completes Discord authorization, but the callback returns an error (e.g.
"redirect_uri mismatch"), or the session never establishes.

Steps:
1. **Immediate:** tell users to use `https://<site>.netlify.app` while the
   fix lands (dual-registration means OAuth still works there).
2. Re-check Discord Developer Portal → OAuth2 → Redirects — both URIs must
   be present and spelled exactly `https://polls.wtcsmapvote.com/auth/callback`
   (no trailing slash mismatch, no `/auth/callback/` variant).
3. Check Supabase dashboard → Authentication → URL Configuration — Site URL
   must match `https://polls.wtcsmapvote.com` (no trailing slash) and the
   Redirect URLs allowlist must include `https://polls.wtcsmapvote.com/*`.
4. If still broken after 15 minutes and the failure is rooted in the
   platform (not a config typo): temporarily revoke the Discord OAuth app
   secret and announce downtime in Discord. Generate a new secret, update
   it in Supabase dashboard → Authentication → Providers → Discord, and
   retry.

Code changes: none needed. The site keeps serving under the Netlify default
URL; the CNAME flip does not need to be reverted. Once Discord + Supabase
config agrees on the prod host, the OAuth flow works on both hosts.

### Contingency B — Netlify HTTPS cert provisioning stalls > 2h for polls.wtcsmapvote.com

Symptoms: Netlify dashboard shows "Certificate: waiting" or "Provisioning
failed" after >2h since the CNAME flip.

Steps:
1. **Do not revert the CNAME.** Let's Encrypt can take up to 24h in rare
   cases and reverting will just reset the clock next time. Netlify docs:
   https://docs.netlify.com/domains-https/https-ssl/troubleshooting-tips/.
2. Click "Renew certificate" in Netlify Domains dashboard.
3. While waiting: serve the app under the Netlify default URL
   (`https://<site>.netlify.app`) and re-link it in Discord as the primary
   (both URIs are already allowlisted from Task 4).
4. Once the cert provisions: return to Task 5 step 3 (prod smoke test).
5. If cert still stalled after 24h: open a Netlify support ticket (free
   tier support is email-only but active).

Code changes: none. The site is functional on the Netlify default URL
throughout; `polls.wtcsmapvote.com` serves once the cert lands.
</rollback>
