# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation & Authentication phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom CSS components) |
| Icon library | lucide-react (tree-shakeable SVG icons) |
| Font | `system-ui, 'Segoe UI', Roboto, sans-serif` (existing `--sans` and `--heading` variables) |

**Source:** STACK.md explicitly chose no component library ("Only ~10 components needed. Custom CSS components are lighter and simpler for this scale."). CSS custom properties already set up in `src/index.css`.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element padding, tight label spacing |
| sm | 8px | Compact element spacing, button icon gap, list item gap |
| md | 16px | Default element spacing, card padding (mobile), form field gap |
| lg | 24px | Section padding, card padding (desktop), nav height padding |
| xl | 32px | Layout gaps between major sections, page horizontal padding (desktop) |
| 2xl | 48px | Major section breaks, landing page section spacing |
| 3xl | 64px | Page-level top/bottom spacing, hero section vertical padding |

Exceptions: Touch targets minimum 44px height (per WCAG 2.5.8) for "Login with Discord" button and nav items on mobile.

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 (24px) | Paragraphs, descriptions, explanatory text |
| Label | 14px | 400 (regular) | 1.4 (20px) | Nav links, meta text, badge text, small UI labels |
| Heading | 20px | 600 (semibold) | 1.3 (26px) | Section titles, card headings, page subtitles |
| Display | 32px | 600 (semibold) | 1.2 (38px) | Landing page hero title, page-level headings |

**Notes:**
- Only 2 font weights are used across the entire project: 400 (regular) and 600 (semibold). Visual hierarchy comes from size contrast, not weight proliferation.
- Mobile body stays at 16px (no reduction below 16px to prevent iOS zoom on input focus).
- Display size reduces to 28px on viewports below 640px.
- Heading size stays 20px across all breakpoints.
- Letter spacing: -0.5px on Display, -0.25px on Heading, 0 on Body and Label.
- The existing 56px h1 and 24px h2 in `src/index.css` will be replaced by this scale. The current CSS is Vite boilerplate being fully replaced in this phase.

## Color

Dark-mode-first palette designed to complement Discord's dark theme while establishing a distinct esports identity. Light mode is NOT implemented in Phase 1 -- dark is the only mode (per CONTEXT.md D-12: "Dark mode should be DEFAULT, matching Discord's dark theme").

| Role | CSS Variable | Value | Usage |
|------|-------------|-------|-------|
| Dominant (60%) | `--bg` | `#0f1117` | Page background, app shell background |
| Surface | `--bg-surface` | `#161820` | Elevated surfaces (cards, dropdowns, modals) |
| Surface raised | `--bg-raised` | `#1c1e28` | Nav bar, hover states on cards, input backgrounds |
| Secondary (30%) | `--bg-secondary` | `#232530` | Active nav items, selected states, code blocks |
| Border | `--border` | `#2a2d3a` | Card borders, dividers, input borders |
| Border hover | `--border-hover` | `#3a3d4a` | Input focus borders, interactive element hover borders |
| Text primary | `--text-primary` | `#e8eaed` | Headings, primary content, nav links |
| Text secondary | `--text-secondary` | `#9ca3af` | Body text, descriptions, secondary labels |
| Text muted | `--text-muted` | `#6b7280` | Placeholder text, disabled states, timestamps |
| Accent (10%) | `--accent` | `#7c5cfc` | Primary CTA buttons, active nav indicator, link text |
| Accent hover | `--accent-hover` | `#6a4ce0` | Button hover state |
| Accent subtle | `--accent-subtle` | `rgba(124, 92, 252, 0.12)` | Accent background tint (badge backgrounds, active tab background) |
| Destructive | `--destructive` | `#ef4444` | Logout button text, error states |
| Destructive subtle | `--destructive-subtle` | `rgba(239, 68, 68, 0.12)` | Error message background |
| Success | `--success` | `#22c55e` | Login success feedback |
| Success subtle | `--success-subtle` | `rgba(34, 197, 94, 0.12)` | Success message background |

**Accent reserved for (explicit list):**
1. "Login with Discord" primary CTA button background
2. Active navigation tab underline/indicator
3. Text links (inline and standalone)
4. User avatar ring on hover (subtle)
5. Loading spinner stroke color

**Accent is NOT used for:** borders in default state, body text, navigation backgrounds, card backgrounds, or decorative elements.

## Component Inventory (Phase 1)

These components must be built in this phase. Each entry describes the visual contract.

| Property | Value |
|----------|-------|

| Content padding (desktop) | 32px horizontal, 32px top below nav |
| Content padding (mobile) | 16px horizontal, 16px top below nav |
| Min height | 100svh |

### 2. Navigation Bar (`NavBar`)

Sticky top bar with logo, navigation links, and user controls.

| Property | Value |
|----------|-------|
| Height | 56px (desktop), 52px (mobile) |
| Background | `--bg-raised` (#1c1e28) |
| Border bottom | 1px solid `--border` |
| Position | `position: sticky; top: 0; z-index: 50` |
| Padding horizontal | 24px (desktop), 16px (mobile) |
| Layout | Flexbox: logo left, nav center, user controls right |

**Logo area:** Text logo "WTCS Polls" in Display weight (600) at 18px, color `--text-primary`. No image logo in Phase 1.

**Nav links (desktop):** Horizontal list: "Polls", "Archive", "Admin" (admin-only, hidden for non-admins). Label typography (14px, 400). Color `--text-secondary` default, `--text-primary` on hover, `--accent` when active. Active indicator: 2px bottom border in `--accent`.

**Mobile navigation:** Hamburger menu button (lucide `Menu`, 24px, `aria-label="Open navigation menu"`) top-right replacing nav links. Tapping opens a slide-down panel below the nav bar, full-width, with nav links stacked vertically. Each link: 48px row height, 16px left padding, `--text-secondary` default, `--bg-secondary` background on active. Panel background: `--bg-surface`. Close on link tap or tap outside.

**User controls (logged in):** Discord avatar (32px circle, `border-radius: 50%`) with username next to it (Label typography, `--text-secondary`). On mobile: avatar only (no username text), 32px. Clicking avatar opens a dropdown menu.

**User dropdown menu:**
- Width: 200px
- Background: `--bg-surface`
- Border: 1px solid `--border`
- Border radius: 8px
- Shadow: `0 4px 16px rgba(0, 0, 0, 0.4)`
- Items: "Sign out" (lucide `LogOut` icon, 16px, color `--destructive`)
- Item padding: 12px 16px
- Item hover: `--bg-secondary` background

### 5. Auth Error Page (`AuthError`)

Full-page error display for 2FA rejection and general auth failures. Chosen over toasts because auth errors are blocking -- the user cannot proceed without resolving the issue, and non-technical esports community members need clear, patient guidance.

**Layout:** Centered single column, max-width 480px, same vertical centering as landing page.

| Element | Specification |
|---------|---------------|
| Icon | lucide `ShieldAlert`, 48px, color `--destructive` |
| Heading | Heading typography (20px, 600), color `--text-primary`. 16px below icon. |
| Body | Body typography (16px, 400), color `--text-secondary`. 8px below heading. |
| Action link/button | Primary button or text link, 24px below body. |
| Background card | `--bg-surface`, 1px `--border` border, 12px radius, 32px padding |

**2FA Rejection (AUTH-02):**
- Heading: "Two-Factor Authentication Required"
- Body: "To keep voting fair for everyone, we require two-factor authentication on your Discord account. It only takes a minute to set up."
- Action: Primary button linking to `https://support.discord.com/hc/en-us/articles/219576828` -- label: "Set Up 2FA on Discord"
- Secondary action: Ghost button "Try Again" that restarts the OAuth flow.

**Session Expired:**
- Heading: "Session Expired"
- Body: "Your login session has ended. This happens automatically for security. Just log in again and you will be right back where you were."
- Action: Primary button "Login with Discord"

**General Auth Error:**
- Heading: "Something Went Wrong"
- Body: "We could not complete your login. This is usually temporary -- please try again in a moment. If it keeps happening, let us know in the Discord server."
- Action: Primary button "Try Again"

## Interaction States

### Auth Flow

| State | What the user sees |
|-------|-------------------|
| Logged out, landing | Landing page with "Login with Discord" CTA |
| OAuth redirect | Browser navigates to Discord OAuth screen (external) |
| Callback processing | Auth callback page with spinner and "Completing login..." |
| Login success | Redirect to Home page, nav shows avatar + username |
| 2FA rejection | Auth error page with 2FA-specific messaging |
| Session expired | Auth error page with session-specific messaging |
| General auth error | Auth error page with generic error messaging |
| Logged in, any page | Nav shows avatar, dropdown available with "Sign out" |
| Sign out | Immediate redirect to landing page (no confirmation dialog) |

### Loading States

| Context | Behavior |
|---------|----------|
| Initial app load (checking session) | Full-page spinner centered in viewport, no flash of landing page |
| OAuth callback | Auth callback page with spinner |
| Navigation between routes | Instant (no loading state -- shell pages are static in Phase 1) |

### Responsive Breakpoints

| Breakpoint | Behavior |
|------------|----------|
| >= 768px (desktop) | Full nav with text links, avatar + username, content max-width 1024px |
| < 768px (mobile) | Hamburger menu, avatar only in nav, full-width content, stacked trust badges |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Color contrast (WCAG AA) | `--text-primary` (#e8eaed) on `--bg` (#0f1117) = 14.5:1. `--text-secondary` (#9ca3af) on `--bg` (#0f1117) = 7.2:1. Both pass AA. |
| Focus indicators | All interactive elements: 2px solid `--accent` outline, 2px offset on `:focus-visible` |
| Touch targets | Minimum 44px height on all tappable elements (buttons, nav links, dropdown items) |
| Keyboard navigation | Tab order follows visual flow. Dropdown menu traps focus while open, Escape closes. |
| Screen reader | Nav uses `<nav>` landmark. Main content uses `<main>`. Buttons have descriptive labels. Avatar dropdown uses `aria-expanded` and `aria-haspopup="menu"`. Hamburger button uses `aria-label="Open navigation menu"`. |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` disables spinner rotation animation and button hover transforms |
