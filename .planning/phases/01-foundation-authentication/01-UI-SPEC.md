# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation & Authentication phase. Built on the WTCS Design System Brief (`.planning/DESIGN-SYSTEM.md`).

---

## Design System

| Property | Value |
|----------|-------|
| Component library | shadcn/ui (Maia style) |
| CSS framework | Tailwind CSS v4 |
| Preset | `bbVJxbc` (Neutral base, Neutral theme, Medium radius) |
| Icon library | Lucide React (ships with shadcn) |
| Font (heading) | Inter (via preset) |
| Font (body) | Inter (via preset) |

**Design direction:** "Warm neutral" — professional but not cold. Linear meets Discord. Clean information density with personality.

**Source:** `.planning/DESIGN-SYSTEM.md` — canonical design brief for all phases.

---

## Spacing

Spacing uses Tailwind's 4px-based default scale. All values are multiples of 4. Key values used in Phase 1:

| Tailwind | Value | Usage |
|----------|-------|-------|
| `p-4` / `gap-4` | 16px | Default element spacing, mobile content padding |
| `p-5` | 20px | Card padding |
| `gap-3` / `space-y-3` | 12px | Card list gap, compact element spacing |
| `space-y-4` | 16px | Inner element gap within cards/sections |
| `px-6` | 24px | Desktop content padding |
| `py-12` / `py-16` | 48px / 64px | Major section spacing, landing page vertical padding |

**Touch targets:** Minimum 44px height on all tappable elements (per WCAG 2.5.8).

**Max width:** `max-w-2xl` (672px) centered — comfortable reading width on desktop, full-width on mobile.

---

## Typography

Uses Tailwind typography utilities. Shadcn Maia preset provides the base scale.

| Role | Tailwind Class | Usage |
|------|---------------|-------|
| Page heading | `text-2xl font-semibold` | Page titles ("Active Topics", "Archive"), auth error headings |
| Section heading | `text-lg font-medium` | Card titles, section headers, empty state headings, nav logo |
| Body | `text-sm text-muted-foreground` | Descriptions, explanatory text, nav links, mobile sheet links |
| Label/meta | `text-xs text-muted-foreground` | Timestamps, response counts, badges, trust row text |

**4 sizes only:** `text-xs` (12px), `text-sm` (14px), `text-lg` (18px), `text-2xl` (24px). Clear visual hierarchy with meaningful steps between each. Result percentages (`text-xl`) deferred to Phase 2 UI-SPEC.

**Constraints:**
- 2 font weights only: `font-medium` (500) and `font-semibold` (600). Regular weight (400) implicit in `text-sm`/`text-xs` without weight class.
- No `text-xs` below 12px. Body stays `text-sm` (14px) minimum.
- No ALL CAPS except category filter pills (`uppercase tracking-wide text-xs`).
- Letter spacing: Default Tailwind values. No custom tracking.

## Color

Uses shadcn theme tokens (CSS custom properties generated by Maia/Neutral preset). Supports light AND dark mode via system preference from day one.

**Color distribution:** `background` dominant (~60%), `card`/`muted` surfaces (~30%), `primary` accent (~10%).

**Shadcn semantic tokens (both modes):**

| Token | Usage |
|-------|-------|
| `background` | Page background |
| `card` | Elevated card surfaces |
| `popover` | Dropdowns, menus |
| `primary` | Primary CTA buttons, active states (muted blue) |
| `primary-foreground` | Text on primary buttons |
| `secondary` | Secondary actions (warm gray) |
| `secondary-foreground` | Text on secondary buttons |
| `muted` | Subtle backgrounds, inactive pill badges |
| `muted-foreground` | Secondary text, descriptions, metadata |
| `accent` | Hover states, subtle highlights (warm neutral) |
| `destructive` | Admin destructive actions only (muted red) |
| `border` | Card borders, dividers |
| `ring` | Focus rings |

**Status colors (for future phases, established now):**

| Status | Light mode | Dark mode |
|--------|-----------|-----------|
| Active/Open | `bg-blue-50 text-blue-700` | `bg-blue-950 text-blue-300` |
| Pinned | Amber banner bar across card top | Amber banner bar |
| Addressed | `bg-green-50 text-green-700` | `bg-green-950 text-green-300` |
| Forwarded | `bg-amber-50 text-amber-700` | `bg-amber-950 text-amber-300` |
| Closed | `bg-neutral-100 text-neutral-600` | `bg-neutral-800 text-neutral-400` |

**Primary accent reserved for:**
1. "Sign in with Discord" CTA button
2. Active navigation indicator
3. Response/opinion submit button (Phase 2+)
4. Links (inline and standalone)

**Primary is NOT used for:** borders in default state, body text, decorative elements.

## Component Inventory (Phase 1)

All components use shadcn/ui primitives where available. Custom components use Tailwind + shadcn design tokens.

### 1. App Shell

Full-viewport container with sticky top nav and scrollable content area.

| Property | Value |
|----------|-------|
| Background | `bg-background` |
| Max content width | `max-w-2xl mx-auto` (672px centered) |
| Content padding (desktop) | `px-6 pt-6` |
| Content padding (mobile) | `px-4 pt-4` |
| Min height | `min-h-svh` |

### 2. Navigation Bar

Sticky top bar. Simple: logo left, nav center/right, user controls right.

**Uses:** Custom component (no shadcn Nav primitive).

| Property | Value |
|----------|-------|
| Background | `bg-card` |
| Border | `border-b` |
| Position | `sticky top-0 z-50` |
| Height | Auto, content-driven with `py-3 px-4 md:px-6` |
| Layout | `flex items-center justify-between` |

**Logo:** Text "WTCS" in `text-lg font-semibold text-foreground`. No image logo in Phase 1.

**Nav links (desktop):** "Topics", "Archive". `text-sm text-muted-foreground hover:text-foreground` with `transition-colors`. Active link: `text-foreground`. No admin link visible to non-admins.

**Mobile navigation:** Hamburger button (lucide `Menu`, `aria-label="Open navigation menu"`). Opens shadcn `Sheet` component (slide-from-top or side) with nav links stacked vertically. Each link: `py-3 text-sm`. Sheet closes on link tap.

**User controls (logged in):** Discord avatar (`rounded-full` 32px) with username (`text-sm text-muted-foreground`, hidden on mobile). Clicking opens shadcn `DropdownMenu`:
- Width: `w-48`
- Items: "Sign out" with lucide `LogOut` icon, `text-destructive`

**User controls (logged out):** "Sign in" link, `text-sm text-muted-foreground hover:text-foreground`.

### 3. Landing Page (logged out)

Centered card-based splash shown to unauthenticated visitors.

**Layout:** `max-w-md mx-auto` centered, `py-16 md:py-24`.

**Uses:** shadcn `Card`, `Button`.

| Element | Specification |
|---------|---------------|
| Card | `bg-card rounded-xl border p-8` |
| Heading | "WTCS Community Suggestions" — `text-2xl font-semibold text-center` |
| Subheading | "Share your opinion on proposals that shape competitive War Thunder." — `text-sm text-muted-foreground text-center mt-2` |

| Explainer | `text-xs text-muted-foreground text-center mt-4` — "Your Discord identity ensures every response is authentic — one account, one opinion." |

### 4. Auth Error Page

Full-page error display using shadcn `Card`. Auth errors are blocking — user cannot proceed without resolving.

**Layout:** `max-w-md mx-auto` centered, `py-16 md:py-24` (same as landing page).

**Uses:** shadcn `Card`, `Button`.

| Element | Specification |
|---------|---------------|
| Card | `bg-card rounded-xl border p-8` |
| Icon | Lucide icon, 40px, `text-destructive` — centered above heading |
| Heading | `text-2xl font-semibold text-center mt-4` |
| Body | `text-sm text-muted-foreground text-center mt-2` |
| Action | shadcn `Button`, `w-full mt-6` |
| Secondary | shadcn `Button` variant="ghost", `w-full mt-2` |

**2FA Rejection (AUTH-02):**
- Icon: `ShieldAlert`
- Heading: "Two-Factor Authentication Required"
- Body: "To keep responses authentic, we require 2FA on your Discord account. It takes about a minute to set up."

- Secondary: "Try Signing In Again" → restarts OAuth flow

**Session Expired:**
- Icon: `Clock`
- Heading: "Session Expired"
- Body: "Your login session has ended. Sign in again to continue."
- Action: "Sign in with Discord"

**General Auth Error:**
- Icon: `AlertCircle`
- Heading: "Something Went Wrong"
- Body: "Could not complete your login. Try again in a moment. If it keeps happening, let us know in the Discord server."
- Action: "Try Signing In Again"

## Interaction States

### Auth Flow

| State | What the user sees |
|-------|-------------------|
| Logged out, landing | Landing card with "Sign in with Discord" CTA |
| OAuth redirect | Browser navigates to Discord OAuth screen (external) |
| Callback processing | Auth callback page with spinner and "Completing login..." |
| Login success | Redirect to Home (or deep link target), nav shows avatar + username |
| 2FA rejection | Auth error card with 2FA-specific messaging |
| Session expired | Auth error card with session-specific messaging |
| General auth error | Auth error card with generic error messaging |
| Logged in, any page | Nav shows avatar, dropdown available with "Sign out" |
| Sign out | Immediate redirect to landing page (no confirmation) |

### Loading States

| Context | Behavior |
|---------|----------|
| Initial app load (checking session) | Full-page spinner centered, no flash of landing page |
| OAuth callback | Auth callback page with spinner |
| Route navigation | Instant (shell pages are static in Phase 1) |

### Responsive Breakpoints

| Breakpoint | Behavior |
|------------|----------|
| `md` and up (>= 768px) | Full nav with text links, avatar + username, `px-6` content padding |
| Below `md` (< 768px) | Hamburger `Sheet` menu, avatar only in nav, `px-4` content padding |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | shadcn Neutral preset meets WCAG AA in both light and dark modes |
| Focus indicators | shadcn default focus rings (`ring-ring`) on all interactive elements |
| Touch targets | Minimum 44px height on all tappable elements (buttons, nav links, dropdown items) |
| Keyboard navigation | Tab order follows visual flow. Sheet/DropdownMenu trap focus while open, Escape closes. |
| Screen reader | Nav uses `<nav>` landmark. Main content uses `<main>`. Hamburger button has `aria-label="Open navigation menu"`. DropdownMenu uses `aria-expanded` and `role="menu"`. |
| Reduced motion | `@media (prefers-reduced-motion: reduce)` disables spinner animation |
| Color mode | Supports light, dark, and system preference. Toggle available in nav. |
