<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client JWT → EF | Non-admin caller must receive 403 (not 200, not 401, not 500) — `requireAdmin` is the authoritative gate |
| EF → audit_log INSERT | Audit write failure must NOT fail the user-facing mutation; helper logs to console.error and continues |
| EF → polls UPDATE | Idempotent: same `hidden` value on consecutive calls must not double-write the audit row |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-EF-ADMIN-BYPASS | Elevation of Privilege | `toggle-results-visibility` EF authn/authz path | mitigate | Re-use the existing `requireAdmin(supabaseAdmin, user.id)` from `_shared/admin-auth.ts` verbatim (no signature change). On `!ok`, return the status from `adminCheckResponse` (403 for non-admin). Plan 04 TEST-12 asserts the 403 for a `memberUser` JWT and the 200 for an `adminUser` JWT. |
| T-AUDIT-DRIFT | Repudiation | Audit row written on every state change | mitigate | EF reads `before.results_hidden` first, UPDATEs, then `if (before !== hidden) writeAudit(...)`. The audit row contains actor_id + before + after JSONB so a flipped state is traceable to its admin. Plan 04 TEST-12 asserts the row presence on state change AND the row absence on no-op (D-11). |
| T-AUDIT-FAIL-OPEN | Repudiation | `writeAudit` Promise<void> contract | accept | If `audit_log` INSERT fails (rare — only on DB issue), the user-facing mutation still returns 200. The audit gap is logged to Supabase Function Logs via `console.error`. Accepted because the alternative (failing the user mutation on audit failure) is a worse UX trade — pre-aggregated vote counts are not exposed by an audit gap. |
| T-IDEMPOTENT-DOUBLELOG | Repudiation | No-op duplicate calls | mitigate | EF pre-reads current state. If `before.results_hidden === hidden`, audit row is NOT written. `results_hidden_changed_at` is still updated (D-13 — timestamp is not gated on state change; cheap under MVCC, makes "last touched by admin" queryable even for no-ops). |
| T-INPUT-VALIDATION | Tampering | EF body parser | mitigate | `poll_id` validated against UUID regex; `hidden` validated as `typeof === 'boolean'` (NOT truthy coercion — `"true"` string and `1` number both 400). JSON parse failure → 400. Body shape derived from pin-poll's existing pattern (unchanged). |
| T-CORS-HEADERS | Information Disclosure | OPTIONS + error responses | mitigate | `getCorsHeaders(req)` called once; threaded through every `json(...)` response including error paths. Verified by analog pin-poll having the same pattern; copied verbatim. |
</threat_model>
