## Spacing Scale

Standard 8-point scale applies. Values referenced in this migration:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon gaps (`mr-1`, `gap-1`) |
| sm | 8px | Inline button group gaps (`gap-2`, `gap-3`) |
| md | 16px | Row internal padding (`p-4`) |
| lg | 24px | CardHeader/CardContent default horizontal padding (`px-6`) — overridden to `px-0` for full-bleed rows |
| xl | 32px | Not used in this migration |
| 2xl | 48px | Not used in this migration |
| 3xl | 64px | Row minimum height (`min-h-[64px]` for admin rows; `min-h-[56px]` for category rows) |

**Exceptions:**
- `CardContent` receives `p-0` class override (D-03) so divide-y lines reach card edges — rows supply their own `p-4` / `min-h-[64px|56px]` internal spacing.
- Admin skeleton rows: `h-[64px]`. Category skeleton rows: `h-[56px]`. Preserved from current code — no change.
- Touch-target minimum for all action buttons: `h-9` (36px) via the existing `size="sm"` Button variant. This is unchanged.

## Typography

All type roles are inherited from the existing design system. No new type scales are introduced.

**Design-chosen weights (2 max):**

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body / Subtext | 14px (`text-sm`) / 12px (`text-xs`) | 400 (regular) | 1.5 / 1.4 | Row labels, secondary text, dialog copy, Discord IDs (monospace), "You" badge |
| Label | 14px (`text-sm`) | 500 (`font-medium`) | 1.5 | Primary row name text (username, category name) |

**Primitive-inherited weight (outside the design palette):**

The `CardTitle` primitive in `src/components/ui/card.tsx` ships `font-semibold` (600) as its own hardcoded className (`leading-none font-semibold`). This is not a design choice introduced by this phase — it is the shadcn new-york CardTitle default. The section heading (`<h2 className="text-base font-semibold">`) in the pre-migration components already renders at 600, so the visual weight is unchanged after migration. Document as: inherited from CardTitle primitive (unchanged). The declared design palette for this phase is exactly 2 weights: 400 + 500.

**Parity verification note:** The executor must confirm via a className/structure diff that no weight class is added or removed from row content elements during migration. Only structural wrapping changes (`<div>` → `<Card>/<CardContent>` etc.) are expected.

## Color

All colors are Tailwind CSS v4 design tokens (CSS custom properties). No new tokens are introduced.

| Role | CSS Token | Light value | Dark value | Usage |
|------|-----------|-------------|------------|-------|
| Dominant (60%) | `--background` | `hsl(220 14% 96%)` | `hsl(20 6% 6%)` | Page background |
| Secondary (30%) | `--card` | `hsl(0 0% 100%)` | `hsl(20 5% 12%)` | Card surface — gains visual pop vs background in both modes |
| Accent (10%) | `--accent` | `hsl(0 0% 96.1%)` | `hsl(20 4% 18%)` | Reserved for: search-result row hover state in PromoteAdminDialog (`hover:bg-accent`) |
| Destructive | `--destructive` | `hsl(0 84.2% 60.2%)` | `hsl(0 62.8% 30.6%)` | Delete icon button text in CategoriesList, destructive Button variant in delete-category Dialog |
| Muted foreground | `--muted-foreground` | `hsl(0 0% 45.1%)` | `hsl(20 3% 60%)` | Discord IDs, "You" badge, empty-state body, skeleton fill (`bg-muted/30`) |
| Border | `--border` | `hsl(0 0% 89.8%)` | `hsl(20 4% 22%)` | Card border (inherited from Card primitive: `border`); divide-y row separators |

**Card border behavior:** The Card primitive ships `rounded-xl border bg-card shadow-sm`. After migration, admin list containers gain `rounded-xl` (was `rounded-md`) and `shadow-sm` (was absent). This is the intentional visual delta per D-01 — consistent with `LandingPage.tsx` and `AuthErrorPage.tsx` Card usage.

**Full-bleed divide-y note:** With `CardContent p-0`, the `divide-y` on the row container reaches the card's left and right edges. The Card's own `border` provides the outer enclosure. Row separators read as internal lines, not as floating sub-borders.

## Component Structure Contract

### UIDN-04: AdminsList.tsx and CategoriesList.tsx

**Target structure (both components share this pattern):**

```
<Card>                                    ← replaces outer <div>
  <CardHeader>                            ← replaces <div className="flex items-center justify-between mb-4">
    <CardTitle>{section name}</CardTitle> ← replaces <h2 className="text-base font-semibold">
    <CardAction>                          ← replaces trailing <Button> in the header row
      <Button size="sm" ...>...</Button>
    </CardAction>
  </CardHeader>
  <CardContent className="p-0">          ← full-bleed override (D-03)
    {/* loading state */}
    <div className="divide-y">
      <div className="h-[64px] bg-muted/30 animate-pulse" data-testid="admin-skeleton" />
      ...
    </div>

    {/* empty state */}
    <div className="flex flex-col items-center justify-center py-16 text-center">
      ...
    </div>

    {/* populated list */}
    <div className="divide-y">
      <div className="flex items-center justify-between p-4 min-h-[64px]">...</div>
      ...
    </div>
  </CardContent>
</Card>
```

**Specific deltas for AdminsList.tsx:**

| Before | After |
|--------|-------|
| `<div>` (outer wrapper) | `<Card>` |
| `<div className="flex items-center justify-between mb-4">` | `<CardHeader>` |
| `<h2 className="text-base font-semibold">Admins</h2>` | `<CardTitle>Admins</CardTitle>` |
| `<Button ...>Promote admin</Button>` (in header) | Wrapped in `<CardAction><Button ...>Promote admin</Button></CardAction>` |
| `<div className="divide-y border rounded-md">` (loading) | `<CardContent className="p-0"><div className="divide-y">` |
| `<div className={admins.length === 0 ? '' : 'divide-y border rounded-md'}>` | `<CardContent className="p-0"><div className="divide-y">` (always rendered; empty state also lives inside CardContent) |
| Empty state: bare `<div className="flex flex-col ...">` | Same `<div>` inside `<CardContent className="p-0">` |

**Specific deltas for CategoriesList.tsx:**

| Before | After |
|--------|-------|
| `<div>` (outer wrapper) | `<Card>` |
| `<div className="flex items-center justify-between mb-4">` | `<CardHeader>` |
| `<h2 className="text-base font-semibold">Categories</h2>` | `<CardTitle>Categories</CardTitle>` |
| `<Button ...>New category</Button>` (in header) | Wrapped in `<CardAction><Button ...>New category</Button></CardAction>` |
| `<div className="divide-y border rounded-md">` (populated) | `<CardContent className="p-0"><div className="divide-y">` |
| `<div className="divide-y">` (loading skeletons — currently lacks `border rounded-md`) | `<CardContent className="p-0"><div className="divide-y">` |
| Empty state: bare `<div className="flex flex-col ...">` | `<CardContent className="p-0"><div className="flex flex-col ...">` |

**Anti-pattern forbidden:** Do NOT wrap individual rows in their own `<Card>`. One `<Card>` per list section only (D-04).

### UIDN-05: PromoteAdminDialog.tsx — Audit Protocol

**Audit question:** Does `<div className="mt-2 border rounded-md divide-y max-h-64 overflow-auto">` (line 88, the search-results container) still constitute a hand-rolled container that warrants Card migration?

**Audit findings from code inspection:**
- The search-results container at line 88 uses `border rounded-md divide-y` — the same hand-rolled pattern UIDN-03 targeted.
- It lives inside a `<Dialog>/<DialogContent>` at nesting level 3 (Dialog → DialogContent → `div.space-y-2` → `div.relative` + results container).
- The `DialogContent` already has `sm:max-w-md` and provides the modal surface. The results container is a sub-section inside the dialog body, not a top-level page section.
- ARIA chain: `<Dialog>` → `<DialogContent role="dialog" aria-labelledby={title-id}>` → `<CardContent>`. A `CardContent` (`<div data-slot="card-content">`) inside DialogContent introduces no new ARIA roles. The dialog's `role="dialog"` and `aria-labelledby` live on `DialogContent` and are unaffected.

**Decision (per D-05):** The flag still applies — hand-rolled `border rounded-md` inside the dialog body is visually inconsistent. Migrate to `<Card>/<CardContent>` for consistency.

**Target structure for PromoteAdminDialog search results block:**

```
{results.length > 0 && (
  <Card className="mt-2 max-h-64 overflow-auto">
    <CardContent className="p-0">
      <div className="divide-y">
        {results.slice(0, 10).map((r) => (
          <div key={r.id} className="flex items-center gap-2 p-2 hover:bg-accent min-h-[44px]">
            ...
          </div>
        ))}
      </div>
    </CardContent>
  </Card>
)}
```

**Changes from current:**
- `<div className="mt-2 border rounded-md divide-y max-h-64 overflow-auto">` → `<Card className="mt-2 max-h-64 overflow-auto"><CardContent className="p-0"><div className="divide-y">`
- The `rounded-md` on each result row (`hover:bg-accent rounded-md`) can be removed — the Card's `rounded-xl` provides the outer shape; inner rows do not need individual rounding.
- `overflow-auto` and `max-h-64` move to the `<Card>` wrapper to retain scroll containment.
- No ARIA changes needed — confirmed no role conflict.

## Snapshot Test Contract

Snapshot diffs from this migration are expected to contain ONLY:

1. `className` changes: `rounded-md` → `rounded-xl`, addition of `shadow-sm`, `bg-card`, `data-slot` attributes from Card primitive internals.
2. Structure additions: `data-slot="card"`, `data-slot="card-header"`, `data-slot="card-title"`, `data-slot="card-action"`, `data-slot="card-content"` wrapper `<div>` elements.
3. Removal of the bare outer `<div>` and the flex header row `<div>`.

Any snapshot diff that shows behavioral changes (event handlers altered, state logic moved, props changed) is a regression and must be investigated before committing.
